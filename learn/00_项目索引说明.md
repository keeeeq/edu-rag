## 🏗️ 项目核心架构

```
用户提问
    ↓
[第0层] BM25精确匹配 (MySQL+Redis) → 匹配到则直接返回 (0.1秒)
    ↓ (未匹配)
[第1层] RAG向量检索 (Milvus) → Hybrid Search + Reranker
    ↓
[第2层] LLM生成答案 (Qwen-Max) → 流式输出
    ↓
返回答案 + 保存到会话历史 (MySQL)
```

**关键设计决策**：
- 三层漏斗架构：BM25拦截70%简单问题，节省LLM成本
- 混合检索：稠密向量(语义) + 稀疏向量(关键词) 互补
- 父子文档切分：子文档检索精准 + 父文档生成完整

## 🔑 核心概念解释

| 术语 | 解释 |
|------|------|
| **RAG** | Retrieval-Augmented Generation，先检索相关文档再让LLM生成答案 |
| **Hybrid Search** | 混合检索，同时使用稠密向量(语义)和稀疏向量(关键词) |
| **BGE-Reranker** | 对检索结果重新精细打分排序的模型 |
| **父子文档切分** | 文档切成小块(子文档)用于检索，保留大块(父文档)用于生成 |
| **HyDE** | 让LLM先生成假设答案，再用假设答案去检索 |
| **BM25** | 经典的关键词匹配算法，用于快速精确匹配 |
